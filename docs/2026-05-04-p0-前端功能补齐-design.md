## 1. 背景与目标

### 1.1 现状

逆熵后端已有完整的 API 体系（18 个路由模块），但前端存在以下缺口：

- **通知系统**：后端有 4 个 notification API，前端零页面/零调用
- **用户资料页**：UserProfile 有 5 个 tab 结构，4 个 tab 显示"功能开发中"
- **导航栏**：App.vue 仅有 `<router-view />`，无全局导航、无用户菜单
- **反馈/举报**：组件已写好但无入口

### 1.2 目标

P0 阶段补齐上述缺口，让前端功能与后端 API 全面匹配。

### 1.3 设计原则

- **最小侵入**：不改动现有页面逻辑，只增加新组件/新入口
- **渐进增强**：先加上导航栏，再逐模块填充内容
- **Mock 同步**：新 API 调用同时补齐 mock 处理

## 2. 需求分解

### 2.1 三个核心模块

| 模块 | 用户需求 | 技术要点 |
|------|---------|---------|
| 通知系统 | 右上角铃铛气泡 → 显示未读数 → 下拉菜单查看通知 → 标记已读 | NotificationBell 组件 + 轮询 + api/notification.js |
| 用户资料页 | 保留 5 个 tab → 4 个占位 tab 填充真实内容 + 1 个反馈入口 | UserInfoDisplay + UserDebates + UserStats + 系统设置 |
| 用户下拉菜单 | 头像/姓名 → 等级 → 个人中心/通知/反馈/退出 | UserDropdownMenu 组件 + 全局 AppLayout |

### 2.2 布局策略

创建 `AppLayout.vue` 提供全局导航栏（含通知铃铛 + 搜索框 + 用户菜单），封装为布局组件。登录态页面使用此布局，Home / Register / ResetPassword 保持原样。

```
AppLayout.vue 布局结构
┌──────────────────────────────────┐
│ [Logo]   [SearchBox]    [🔔][👤] │  ← 顶部导航栏
├──────────────────────────────────┤
│         <router-view />         │  ← 页面内容
└──────────────────────────────────┘
```

## 3. 模块一：通知系统 (NotificationBell)

### 3.1 新增文件

| 文件 | 内容 |
|------|------|
| `frontend/src/api/notification.js` | 4 个 API 调用封装 |
| `frontend/src/components/NotificationBell.vue` | 铃铛组件 + 下拉面板 |

### 3.2 API 封装

```javascript
// api/notification.js
export const getNotifications = (params) => request.get('/api/notification', { params })
export const getUnreadCount = () => request.get('/api/notification/unread-count')
export const markAsRead = (id) => request.put(`/api/notification/${id}/read`)
export const markAllAsRead = () => request.put('/api/notification/read-all')
```

### 3.3 NotificationBell 组件行为

```
[🔔 3]  ← 铃铛图标 + 未读红色数字气泡
   ↓ 点击
┌─────────────────────┐
│ 通知                │
│─────────────────────│
│ 📣 您的辩论已结算    │  → 点击标记已读 + 跳转
│    2分钟前           │
│─────────────────────│
│ 📣 有用户回复了您    │
│    10分钟前          │
│─────────────────────│
│ [全部标记已读]       │
└─────────────────────┘
```

**状态管理：**
- `unreadCount`: ref(0)，onMounted 时从 `/api/notification/unread-count` 拉取
- `notifications`: ref([])，每次展开时从 `/api/notification` 拉取最新 10 条
- 轮询：`setInterval(getUnreadCount, 30000)` 更新未读数
- `onUnmounted` 清除定时器

**通知类型显示：**
| type | 图标 | 说明 |
|------|------|------|
| debate_settled | 📣 | 辩论已结算 |
| debate_reply | 💬 | 有人回复了我的发言 |
| vote_result | 🗳 | 投票结果 |
| system | 📢 | 系统通知 |
| punishment | ⚠️ | 处罚通知 |

**Props / State:**
- `debates`: ref([]) — 分页列表

- 状态标签：0=待开始(黄), 1=进行中(绿), 2=已结束(灰)

```
┌─────────────────────────────────────────────┐
│ [逆熵]  [SearchBox]          [🔔][头像 ▼]  │  ← 固定顶部

│                                             │
│          <router-view />                    │  ← 页面内容
```

- 仅在登录态显示（未登录时不出现，维持现在的 Home 样式）

### 5.2 UserDropdownMenu 组件

```
[头像] [用户名] ▼
    ↓ 点击
┌─────────────────┐
│ 👤 用户名        │
│     Lv.3 资深级  │
│─────────────────│
│ ❖ 个人中心      │
│ 🔔 通知中心      │
│ ✉ 意见反馈      │
│─────────────────│
│ 🚪 退出登录      │
└─────────────────┘
```

**行为：**
- 点击切换下拉面板显隐
- 点击外部区域关闭面板
- "个人中心" → 跳转到 `/profile`
- "通知中心" → 触发 NotificationBell 展开下拉面板（复用已有通知组件）
- "意见反馈" → 打开 FeedbackForm 弹窗
- "退出登录" → 清除 token + user，跳转到 Home

### 5.3 页面接入布局

**策略：不修改各页面文件，在 App.vue 中条件渲染**

AppLayout 通过 `<slot />` 接收页面内容：

```vue
<template>
  <div class="app-layout">
    <nav class="nav-bar">
      <div class="nav-left">
        <router-link to="/debates" class="logo">逆熵</router-link>
        <SearchBox />
      </div>
      <div class="nav-right">
        <NotificationBell />
        <UserDropdownMenu />
      </div>
    </nav>
    <main class="main-content">
      <slot />
    </main>
  </div>
</template>
```

在路由 meta 中添加 `requiresLayout` 标记：

```javascript
// router/index.js
{
  path: '/debates',
  name: 'DebateList',
  component: () => import('@/views/DebateList.vue'),
  meta: { requiresLayout: true }
}
```

App.vue 按此条件渲染：

```vue
<template>
  <div id="app">
    <router-view v-if="!route.meta.requiresLayout" />
    <AppLayout v-else>
      <router-view />
    </AppLayout>
  </div>
</template>
```

**需要 `requiresLayout: true` 的路由：**
- `/debates`, `/debates/create`, `/debates/flow`
- `/rules`, `/rules/debate/:id`, `/rules/history`
- `/profile`
- `/admin`
- `/check/logic`, `/check/debate`, `/check/result`

**保持原样（无 `requiresLayout`）的路由：**
- `/` (Home)
- `/register`
- `/reset-password`

## 9. 边界与风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 通知轮询可能增加服务端压力 | 次要 | 轮询间隔 30s，只请求 unread-count（轻量查询） |
| AppLayout 可能影响现有页面样式 | 中等 | 使用固定定位 + z-index，页面内容 padding-top 留空间 |
| 未登录页面需要判断显示 | 中等 | router meta 标记 `requiresLayout`，AppLayout 内判断 |
| 现有 CSS 变量命名不一致 | 低 | 使用 Element Plus 风格，不混用自定义变量 |
