## 1. 背景与目标

### 1.1 现状

逆熵现有的辩论是"自由模式"：用户创建话题 → 选择立场 → 自由发言 → 投票 → 结算。没有轮次概念，没有时间限制，没有发言顺序约束。

### 1.2 目标

引入**辩论模板** 系统，让辩论从"发帖式讨论"升级为"有规则、有轮次、有评分"的竞技式辩论。

### 1.3 首发模板：1v1 标准赛

- 正反各 1 人
- 7 个固定轮次，按序发言
- 每轮有限时（5 分钟/4 分钟/3 分钟）
- 超时自动跳过
- 结束后双方和观众按 4 项标准评分

## 2. 数据模型

### 2.1 模板表 `debate_template`

```sql
CREATE TABLE `debate_template` (
  `id` INT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '模板ID',
  `name` VARCHAR(100) NOT NULL COMMENT '模板名称，如"1v1 标准赛"',
  `type` VARCHAR(50) NOT NULL COMMENT '模板标识，如 standard_1v1',
  `description` VARCHAR(500) DEFAULT '' COMMENT '模板简要说明',
  `config` JSON NOT NULL COMMENT '轮次配置 + 评分标准 JSON',
  `is_active` TINYINT(1) NOT NULL DEFAULT 1 COMMENT '是否可用',

  `updated_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  UNIQUE KEY `uk_type` (`type`)
```

### 2.2 轮次记录表 `debate_round`

```sql
CREATE TABLE `debate_round` (
  `id` INT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '轮次ID',
  `topic_id` INT UNSIGNED NOT NULL COMMENT '关联辩论话题',
  `round_order` TINYINT UNSIGNED NOT NULL COMMENT '轮次序号（从1开始）',
  `round_name` VARCHAR(100) NOT NULL COMMENT '轮次名称',
  `speaker_stance` TINYINT NOT NULL COMMENT '应该谁发言：1=正方, 0=反方',
  `speaker_id` INT UNSIGNED DEFAULT NULL COMMENT '实际发言者ID（超时跳过时为NULL）',
  `content` TEXT DEFAULT NULL COMMENT '本轮发言内容',
  `duration_sec` INT UNSIGNED NOT NULL COMMENT '本轮的限定时间（秒）',
  `used_sec` INT UNSIGNED DEFAULT NULL COMMENT '实际用时（秒）',
  `status` ENUM('waiting','active','completed','timeout','skipped') NOT NULL DEFAULT 'waiting' COMMENT '轮次状态',
  `started_at` TIMESTAMP NULL DEFAULT NULL COMMENT '本轮开始时间',
  `ended_at` TIMESTAMP NULL DEFAULT NULL COMMENT '本轮结束时间',
  `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_topic_round` (`topic_id`, `round_order`),
  KEY `idx_speaker` (`speaker_id`),
  CONSTRAINT `fk_round_topic` FOREIGN KEY (`topic_id`) REFERENCES `debate_topic` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='辩论轮次记录';
```

### 2.3 评分表 `debate_score`

```sql
CREATE TABLE `debate_score` (
  `id` INT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '评分ID',
  `topic_id` INT UNSIGNED NOT NULL COMMENT '关联辩论话题',
  `voter_id` INT UNSIGNED NOT NULL COMMENT '评分者ID',
  `target_stance` TINYINT NOT NULL COMMENT '评分对象：1=正方, 0=反方',
  `criterion_key` VARCHAR(50) NOT NULL COMMENT '评分项标识（对应模板 scoring.criteria[].key）',
  `score` TINYINT UNSIGNED NOT NULL COMMENT '得分（1-10）',
  `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_vote` (`topic_id`, `voter_id`, `target_stance`, `criterion_key`),
  KEY `idx_topic` (`topic_id`),
  KEY `idx_voter` (`voter_id`),
  CONSTRAINT `fk_score_topic` FOREIGN KEY (`topic_id`) REFERENCES `debate_topic` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='辩论评分记录';
```

### 2.4 `debate_topic` 表改动

在 `debate_topic` 表中增加字段：

```sql
ALTER TABLE `debate_topic`
```

## 5. 轮次控制引擎

### 5.1 核心函数 `advanceRound(topicId, conn)`

这是整个系统的核心逻辑，在每次轮次变更时调用：

```
advanceRound(topicId, conn):
  1. 查询 debate_topic 的当前轮次 currentRound
  2. 根据 template_id 获取模板 config
  3. 计算下一轮序号 = currentRound + 1
  4. 如果下一轮 > 总轮数:
     a. 将 topic status 设为 CLOSED（投票中）
     b. 返回 { debateComplete: true }
  5. 否则:
     a. 从 config.rounds 获取下一轮定义
     b. 创建新的 debate_round 记录（status='active', started_at=now）
     c. 更新 debate_topic.current_round = 下一轮序号
     d. 返回 { nextRound: {...} }
```

### 5.2 发言提交流程

```
submitRoundSpeech(topicId, roundId, userId, content):
  1. 获取 debate_round 记录
  2. 校验：
     a. round.status === 'active'
     b. userId 与 round.speaker_id 匹配
     c. 用户是 topic 的参与者（已在 debate_participant 中）
  3. 计算 used_sec = now - round.started_at
  4. 如果 used_sec > durationSec:
     a. 标记为 timeout
     b. 调用 advanceRound()
     c. 返回"已超时"
  5. 否则:
     a. 执行 aiAudit 审核内容
     b. 更新 round（content, status='completed', used_sec, ended_at）
     c. 调用 advanceRound()
     d. 返回下一轮或完成信号
```

### 5.3 超时自动跳过

超时自动跳过由前端计时器触发：

```
1. 前端 DebateRoundTimer 组件每秒检查 remainingSec
2. 当 remainingSec <= 0 时：
   a. 调用 POST /skip
   b. 后端校验当前 round 是否仍为 active
   c. 如果是，标记为 'timeout'，speaker_id=null
   d. 调用 advanceRound()
```

### 5.4 自由辩论的特殊性

第 5 轮"自由辩论"的正反方划分做一个特殊处理：

- 模板配置中 `speaker: "pro"` 只是标记正方先开始
- 自由辩论模式下：
  - 正方提交发言后会变为"轮到反方"
  - 反方提交后回到正方
  - 每方在上次发言后 15 秒冷却后才可再次发言（防刷屏）
  - 但总时长限制为 3 分钟（180 秒）
  - 双方都可以在总时间内发言

具体实现：自由辩论轮的 `speaker_stance` 在每次提交后翻转一次（1→0→1→0…）。

## 6. 评分系统

### 6.1 评分流程

```
辩论结束（debate_topic.status = CLOSED）
  → 进入评分阶段
  → 所有参与者（正反方 + 观众）均可评分
  → 每人对正反双方各打 4 项分（共 8 次评分）
  → 每个 (topic, voter, target_stance, criterion) 唯一，可修改重评
  → 评分阶段持续 48 小时（可配置）
  → 到期后自动结算，显示最终结果
```

### 6.2 计分规则

```
每人给正方评分：
  argument(8) + evidence(7) + rebuttal(9) + expression(8) = 32

每人给反方评分：
  argument(7) + evidence(6) + rebuttal(7) + expression(7) = 27

最终正方总分 = 所有评分者的正方总分之和 / 评分人数
最终反方总分 = 所有评分者的反方总分之和 / 评分人数

评分者权重（复用 USER_LEVEL 体系）：
  BEGINNER(1): 1.0x
  INTERMEDIATE(2): 1.0x
  ADVANCED(3): 1.2x
  ADMIN(4): 1.5x
```

### 6.3 评分 API 返回

```json

{

    "scores": {
      "pro": {

```

- 正反方参与者：基础 50 exp + (获胜方额外 30 exp)
- 评分观众：每人 10 exp（限制每场辩论只奖一次）

## 8. 边界与风险

### 8.1 已知边界

| 场景 | 处理方式 |
|------|---------|
| 用户创建辩论时选了模板，但没人加入 | 模板不起作用，话题保持 DRAFT |
| 只有一人加入 | 模板辩论无法开始（需双方就位），等待 |
| 两人都已加入，但其中一人退出 | 轮次推进到该方发言时自动跳过 |
| 网络断连导致倒计时不准确 | 后端 `submitRound` 校验时以服务器时间计算 `used_sec`，前端计时仅做 UI 提示 |
| 自由辩论轮次谁也抢不着 | 冷却期 15 秒，防刷屏 |

### 8.2 风险与缓解

| 风险 | 级别 | 缓解措施 |
|------|------|---------|
| 轮次推进在并发请求下产生竞态条件 | **高** | `advanceRound` 中加事务 + SELECT FOR UPDATE |
| 自由辩论的 speaker 翻转逻辑意外 | 中 | 专门对 free_debate 模式写独立测试 |
| 评分阶段无人评分 | 低 | 自动在 48 小时后以现有投票权重结算为保底 |
| 超时跳过和用户提交几乎同时触发 | 中 | skip 和 submit 共用事务锁，先到者生效 |
