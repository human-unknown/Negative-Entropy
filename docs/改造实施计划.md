### 1.1 新增 SQL 迁移

```
backend/sql/
```

#### `31_post.sql` — 帖子表

```sql
CREATE TABLE post (
  id            BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  channel_id    INT UNSIGNED NOT NULL,
  author_id     BIGINT UNSIGNED NOT NULL,
  title         VARCHAR(200) NOT NULL,
  content       TEXT NOT NULL,
  -- 论证结构（可选但鼓励）
  thesis        VARCHAR(500) NULL COMMENT '核心论点（一句话）',
  premises      JSON NULL COMMENT '前提/论据列表',
  -- 来源引用
  sources       JSON NULL COMMENT '[{url, title, note}]',
  -- 状态
  status        TINYINT NOT NULL DEFAULT 0 COMMENT '0=正常 1=关闭 2=已删除',
  audit_status  TINYINT NOT NULL DEFAULT 0 COMMENT '0=待审 1=通过 2=拒绝',
  -- 互动统计（冗余加速）
  comment_count INT UNSIGNED NOT NULL DEFAULT 0,
  view_count    INT UNSIGNED NOT NULL DEFAULT 0,
  quality_score DECIMAL(3,2) NULL COMMENT '综合质量分 0-10',
  -- 辩论关联（可选）
  debate_id     BIGINT UNSIGNED NULL COMMENT '关联的辩论话题ID',
  -- 元数据
  is_pinned     TINYINT(1) NOT NULL DEFAULT 0,
  created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  INDEX idx_channel (channel_id),
  INDEX idx_author (author_id),
  INDEX idx_status_created (status, created_at),
  INDEX idx_quality (quality_score),
  FOREIGN KEY (channel_id) REFERENCES channel(id),
  FOREIGN KEY (author_id) REFERENCES user(id),
  FOREIGN KEY (debate_id) REFERENCES debate_topic(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

#### `32_comment.sql` — 评论表

```sql
CREATE TABLE comment (
  id            BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  post_id       BIGINT UNSIGNED NOT NULL,
  author_id     BIGINT UNSIGNED NOT NULL,
  parent_id     BIGINT UNSIGNED NULL COMMENT '父评论ID（嵌套回复）',
  content       TEXT NOT NULL,
  -- 论证相关
  sources       JSON NULL COMMENT '[{url, title, note}]',
  -- 状态
  audit_status  TINYINT NOT NULL DEFAULT 0 COMMENT '0=待审 1=通过 2=拒绝',
  is_deleted    TINYINT(1) NOT NULL DEFAULT 0,
  -- 互动
  upvote_count  INT UNSIGNED NOT NULL DEFAULT 0,
  -- 元数据
  created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,

  INDEX idx_post (post_id),
  INDEX idx_author (author_id),
  INDEX idx_parent (parent_id),
  FOREIGN KEY (post_id) REFERENCES post(id),
  FOREIGN KEY (author_id) REFERENCES user(id),
  FOREIGN KEY (parent_id) REFERENCES comment(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 1.2 需要修改的现有表

#### `debate_topic` — 新增关联字段

```sql
ALTER TABLE debate_topic
  ADD COLUMN post_id BIGINT UNSIGNED NULL AFTER id,
  ADD INDEX idx_post (post_id),
  ADD FOREIGN KEY (post_id) REFERENCES post(id) ON DELETE SET NULL;
```

> 这样辩论可以从帖子衍生，也可以独立存在（兼容旧数据）。

#### `user` — 新增权限字段（可选）

```sql
ALTER TABLE user
  ADD COLUMN can_post TINYINT(1) NOT NULL DEFAULT 0 COMMENT '通过逻辑测试后设为1';
```

> 或直接用 `user_check.status = 'passed'` 判断，不新增字段。后者更推荐（避免冗余）。

### 2.3 API 端点设计

#### 频道 `/api/channels`

| POST | `/api/channels` | 创建频道 | 管理员 |

#### 帖子 `/api/posts`

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/posts` | 帖子列表（首页信息流） | 无 |
| GET | `/api/posts/:postId` | 帖子详情 | 无 |
| POST | `/api/posts` | 发帖 | Lv2+（通过逻辑测试） |
| PUT | `/api/posts/:postId` | 编辑帖子 | 作者本人 |
| DELETE | `/api/posts/:postId` | 删除帖子 | 作者/管理员 |
| POST | `/api/posts/:postId/pin` | 置顶 | 管理员 |
| POST | `/api/posts/:postId/debate` | 升级为辩论 | Lv3+ |

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/posts/:postId/comments` | 评论列表（树形） | 无 |

| DELETE | `/api/comments/:commentId` | 删除评论 | 作者/管理员 |

#### 评分 `/api/posts`

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/api/posts/:postId/score` | 质量评分 | Lv2+ |
| POST | `/api/comments/:commentId/upvote` | 点赞评论 | Lv2+ |

## 第四阶段：权限体系重构

### 4.1 现有等级 → 新权限映射

| 等级 | 名称 | 获得方式 | 新权限 |
|------|------|---------|--------|
| Lv1 | 访客 | 注册 | 浏览、点赞 |
| Lv2 | 公民 | 通过逻辑测试 | **发帖、评论、评分** |
| Lv3 | 学者 | 500经验 | **开启辩论**、写长文、编辑标签 |
| Lv4 | 裁判 | 5000经验 / 选举 | 审核内容、管理频道 |

### 4.2 逻辑测试定位调整

- **当前**：通过逻辑测试 = 可以参与辩论
- **改造后**：通过逻辑测试 = Lv1→Lv2 晋升 = **可以发帖和评论**

这是关键改变——逻辑测试从"辩论准入"变成"社区发言准入"。

### 4.3 前端权限控制

```js
// utils/permission.js — 新增
export function canPost(level) {
  return level >= USER_LEVEL.INTERMEDIATE;  // Lv2+
}

export function canComment(level) {
  return level >= USER_LEVEL.INTERMEDIATE;  // Lv2+
}

export function canStartDebate(level) {
  return level >= USER_LEVEL.ADVANCED;      // Lv3+
}
```

## 风险与决策点

### 需要确认的问题

1. **旧辩论数据怎么处理？** 现有的 `debate_topic` 数据是独立存在的，改造后 `post_id` 可为 NULL，旧数据保留。是否要给所有旧辩论自动生成对应的帖子？

2. **首页要不要未登录游客可见？** 信息流是否对未登录用户开放？建议：帖子列表可见（吸引注册），发帖/评论需登录+逻辑测试。

3. **质量评分谁来做？** 当前辩论评分是 Lv2+ 用户打分。帖子和评论的评分是所有人都能评，还是也限定 Lv2+？建议：限定 Lv2+（通过逻辑测试的用户才有评分资格）。

4. **Mock 模式保留还是删除？** 建议保留 Mock 作为离线开发模式，但不作为默认值。
