# 举报申诉流程联调指南

## 流程说明

完整流程：**用户举报 → 管理员复核 → 处罚 → 用户申诉 → 复核结案**

## 修复内容

### 1. 新增数据表
- [`backend/sql/24_report.sql`](backend/sql/24_report.sql) - 举报表
- [`backend/sql/25_notification.sql`](backend/sql/25_notification.sql) - 通知表

### 2. 修复的Bug
- ✅ 修复用户ID字段不一致（`req.user.id` → `req.user.userId`）
- ✅ 修复状态同步问题（申诉通过后正确更新用户状态为 `'normal'`）
- ✅ 添加事务回滚保护，防止流程阻塞
- ✅ 添加通知机制，用户收到处罚和申诉结果通知

### 3. 新增功能
- [`reviewReport()`](backend/src/controllers/reportController.js:45) - 管理员审核举报
- [`getPendingReports()`](backend/src/controllers/reportController.js:107) - 获取待审核举报列表
- [`getPendingAppeals()`](backend/src/controllers/appealController.js:67) - 获取待审核申诉列表

## API测试流程

### 步骤1：用户提交举报
```bash
POST /api/report
Authorization: Bearer <user_token>
Content-Type: application/json

{
  "target_type": "user",
  "target_id": 2,
  "type": "辱骂",
  "reason": "该用户在辩论中使用不当言论攻击他人"
}
```

### 步骤2：管理员查看待审核举报
```bash
GET /api/report/pending
Authorization: Bearer <admin_token>
```

### 步骤3：管理员审核举报并处罚
```bash
PUT /api/report/:reportId/review
Authorization: Bearer <admin_token>
Content-Type: application/json

{
  "action": "approve",
  "punishType": 2,
  "duration": 7,
  "reason": "确认违规，禁言7天"
}
```

**处罚类型：**
- `1` - 警告
- `2` - 禁言
- `3` - 封号

### 步骤4：用户提交申诉
```bash
POST /api/appeal
Authorization: Bearer <user_token>
Content-Type: application/json

{
  "punishId": 1,
  "reason": "我认为这是误判，我的言论并未违反规则，请重新审核"
}
```

### 步骤5：管理员查看待审核申诉
```bash
GET /api/appeal/pending
Authorization: Bearer <admin_token>
```

### 步骤6：管理员审核申诉
```bash
PUT /api/appeal/:appealId/review
Authorization: Bearer <admin_token>
Content-Type: application/json

{
  "status": 1,
  "result": "经复核，确认误判，撤销处罚"
}
```

**审核状态：**
- `1` - 通过（撤销处罚）
- `2` - 驳回（维持处罚）

## 状态流转

### 举报状态
- `0` - 待审核
- `1` - 已处理（通过）
- `2` - 已驳回

### 申诉状态
- `0` - 待审核
- `1` - 通过（撤销处罚）
- `2` - 驳回（维持处罚）

### 用户状态
- `'normal'` - 正常
- `'muted'` - 禁言
- `'banned'` - 封号

## 关键修复点

1. **用户ID统一**：所有控制器使用 `req.user.userId`
2. **状态同步**：申诉通过后用户状态更新为 `'normal'` 字符串
3. **事务完整性**：所有数据库操作使用事务，失败自动回滚
4. **通知机制**：处罚和申诉结果自动通知用户
5. **权限校验**：管理员接口使用 [`requireAdmin`](backend/src/middlewares/permission.js) 中间件
