# 投票结算流程测试指南

## 📋 测试目标

联调投票结算完整流程：**辩论结束 → 暗投 → 票数统计 → 胜负判定 → 结果展示**

## 🔧 修复内容

### 1. 权重计算修复
- ✅ 添加用户等级权重系统
  - 普通用户：1.0 权重
  - 资深用户：1.5 权重
  - 管理员：2.0 权重
- ✅ 投票时记录用户ID和权重
- ✅ 结算时使用加权票数统计

### 2. 平票判定修复
- ✅ 修复平票判定逻辑（使用浮点数比较）
- ✅ 平票时 winner 为 NULL
- ✅ 平票时显示"等待管理员评审"

### 3. 统计错误修复
- ✅ 使用 `SUM(weight)` 替代 `COUNT(*)`
- ✅ 返回加权票数而非简单计数
- ✅ 结果展示显示加权票数

### 表结构变更

**旧表结构：**
```sql
CREATE TABLE debate_vote (
  id BIGINT,
  topic_id BIGINT,
  stance TINYINT,
  created_at DATETIME
);
```

**新表结构：**
```sql
CREATE TABLE debate_vote (
  id BIGINT,
  topic_id BIGINT,
  user_id BIGINT,        -- 新增
  stance TINYINT,
  weight DECIMAL(3,1),   -- 新增
  created_at DATETIME
);
```

## 🧪 测试步骤

### 1. 访问测试页面

```
http://localhost:5173/test/vote-settle
```

### 2. 完整流程测试

#### 步骤1: 结束辩论
1. 点击"结束辩论"按钮
2. 验证状态从"进行中"变为"已结束"
3. 自动进入投票环节

#### 步骤2: 暗投环节
1. 选择支持的立场（正方/反方）
2. 点击"确认投票"
3. 验证投票成功提示
4. 验证投票结果不公开显示

#### 步骤3: 票数统计
1. 点击"执行结算"
2. 后端统计加权票数
3. 验证统计数据正确

#### 步骤4: 胜负判定
1. 系统自动判定胜负
2. 验证判定逻辑：
   - 正方加权票数 > 反方 → 正方获胜
   - 反方加权票数 > 正方 → 反方获胜
   - 加权票数相等 → 平票

#### 步骤5: 结果展示
1. 查看结果面板
2. 验证显示内容：
   - 获胜方标识
   - 加权票数对比
   - 平票时的管理员评审提示

## 🔍 测试场景

### 场景1: 正方获胜
- 正方票数：3票（权重：1.0 + 1.5 + 2.0 = 4.5）
- 反方票数：2票（权重：1.0 + 1.0 = 2.0）
- 预期结果：正方获胜，4.5:2.0

### 场景2: 反方获胜
- 正方票数：2票（权重：1.0 + 1.0 = 2.0）
- 反方票数：2票（权重：1.5 + 2.0 = 3.5）
- 预期结果：反方获胜，3.5:2.0

### 场景3: 平票
- 正方票数：2票（权重：1.0 + 2.0 = 3.0）
- 反方票数：2票（权重：1.5 + 1.5 = 3.0）
- 预期结果：平票，等待管理员评审

## ⚠️ 注意事项

1. **权限要求**
   - 结束辩论：仅话题发布者
   - 投票：所有用户（一人一票）
   - 结算：仅话题发布者

2. **状态检查**
   - 投票前必须先结束辩论
   - 结算前必须有投票数据
   - 不能重复投票
   - 不能重复结算

3. **权重规则**
   - 普通用户（level < 2）：1.0
   - 资深用户（level >= 2）：1.5
   - 管理员（level = 4）：2.0

## 🐛 已知问题修复

### 问题1: 投票不记录用户信息
**修复前：** 只记录 stance，无法追踪用户
**修复后：** 记录 user_id 和 weight

### 问题2: 统计使用简单计数
**修复前：** `COUNT(*)` 简单计数
**修复后：** `SUM(weight)` 加权统计

### 问题3: 平票判定错误
**修复前：** 整数比较可能出错
**修复后：** 浮点数比较，正确处理平票

## 📝 API 端点

```
POST /api/debates/topics/:topicId/vote
Body: { stance: 1 | 2 }
```

### 结算
```
POST /api/debates/topics/:topicId/settle
Response: {
  pro_votes: number,
  con_votes: number,
  winner: 1 | 2 | null,
  summary: string
}
```
