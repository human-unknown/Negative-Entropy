# 权限校验修复说明

## 修复概述

本次修复针对系统中的权限漏洞进行了全面的强化，确保入门/进阶/资深/管理员四个等级的权限严格隔离，防止越权访问和越权操作。

## 用户等级定义

- **入门级 (BEGINNER = 1)**: 基础用户，可参与辩论
- **进阶级 (INTERMEDIATE = 2)**: 进阶用户，可参与辩论
- **资深级 (ADVANCED = 3)**: 资深用户，可参与辩论 + 发布话题
- **管理员级 (ADMIN = 4)**: 管理员，拥有全部权限

## 后端修复内容

### 1. 权限中间件增强 ([`backend/src/middlewares/permission.js`](backend/src/middlewares/permission.js:1))

新增功能：
- 添加 [`requireLevel(minLevel)`](backend/src/middlewares/permission.js:38) 中间件：检查用户是否达到最低等级要求
- 增强 [`requireAdmin`](backend/src/middlewares/permission.js:63) 中间件：使用常量 `USER_LEVEL.ADMIN` 替代硬编码
- 所有中间件在验证成功后将用户等级附加到 `req.user.level`

### 2. 辩论路由权限修复 ([`backend/src/routes/debateRoutes.js`](backend/src/routes/debateRoutes.js:1))

修复的权限漏洞：
- [`POST /topics`](backend/src/routes/debateRoutes.js:15): 创建话题 - 添加 `requireLevel(USER_LEVEL.ADVANCED)` 校验
- [`PUT /speeches/:speechId/audit`](backend/src/routes/debateRoutes.js:23): 审核发言 - 添加 `requireAdmin` 校验

### 3. 规则辩论路由权限修复 ([`backend/src/routes/ruleDebateRoutes.js`](backend/src/routes/ruleDebateRoutes.js:1))

修复的权限漏洞：
- [`POST /`](backend/src/routes/ruleDebateRoutes.js:9): 创建规则辩论 - 添加 `requireAdmin` 校验
- [`POST /:debateId/settle`](backend/src/routes/ruleDebateRoutes.js:13): 结算规则辩论 - 添加 `requireAdmin` 校验

### 4. 用户路由权限修复 ([`backend/src/routes/userRoutes.js`](backend/src/routes/userRoutes.js:1))

修复的权限漏洞：
- [`POST /add-exp`](backend/src/routes/userRoutes.js:8): 添加经验值 - 添加 `requireAdmin` 校验（防止用户自行刷经验）

### 5. 控制器冗余检查清理

移除了控制器中的冗余权限检查，统一在路由层面进行权限校验：
- [`debateController.js`](backend/src/controllers/debateController.js:1): 移除 `createTopic`、`auditTopic`、`auditSpeech` 中的重复权限检查
- [`ruleDebateController.js`](backend/src/controllers/ruleDebateController.js:1): 移除 `createRuleDebate`、`settleRuleDebate` 中的重复权限检查
- 统一 userId 获取方式：`req.user?.userId || req.user?.id`

## 前端修复内容

### 1. 路由守卫增强 ([`frontend/src/router/index.js`](frontend/src/router/index.js:1))

增强的校验逻辑：
- 严格校验：需要认证的路由必须同时有 token 和 user 信息
- 管理员权限严格校验：必须是 `USER_LEVEL.ADMIN`
- 等级权限严格校验：用户等级必须 >= `minLevel`
- 权限不足时输出警告日志并重定向
- 认证失败时清除本地存储的 token 和 user

新增路由权限标记：
- [`/profile`](frontend/src/router/index.js:23): 需要入门级及以上
- [`/debates/flow`](frontend/src/router/index.js:54): 需要入门级及以上
- [`/test/vote-settle`](frontend/src/router/index.js:59): 需要入门级及以上
- [`/admin`](frontend/src/router/index.js:64): 需要管理员权限

### 2. API 请求拦截器增强 ([`frontend/src/api/request.js`](frontend/src/api/request.js:1))

增强的错误处理：
- 401 未认证：清除 token 和 user，重定向到首页
- 403 权限不足：输出错误日志，拒绝请求
- 统一的权限错误处理机制

## 安全加固措施

### 后端
1. **双重校验**：路由层 + 中间件层权限校验
2. **统一错误码**：401 未认证、403 权限不足
3. **数据库实时查询**：每次请求都从数据库查询最新用户等级
4. **防止硬编码**：使用常量定义等级，避免魔法数字

### 前端
1. **路由守卫**：访问页面前进行权限校验
2. **API 拦截器**：请求失败时统一处理权限错误
3. **本地存储清理**：认证失败时清除敏感信息
4. **日志记录**：权限不足时输出警告日志便于调试

## 测试建议

### 越权访问测试
1. 使用入门级账号尝试访问 `/debates/create` 页面
2. 使用进阶级账号尝试访问 `/admin` 页面
3. 使用资深级账号尝试访问管理员接口

### 越权操作测试
1. 入门级用户尝试调用 `POST /api/debates/topics` 创建话题
2. 资深级用户尝试调用 `PUT /api/debates/topics/:id/audit` 审核话题
3. 非管理员用户尝试调用 `POST /api/user/add-exp` 添加经验值
4. 非管理员用户尝试调用 `POST /api/rule-debates` 创建规则辩论

### 预期结果
- 前端：权限不足时重定向到首页或话题列表
- 后端：返回 403 状态码和"权限不足"错误信息
- 控制台：输出权限不足的警告日志

## 注意事项

1. **等级升级**：用户等级升级后需要重新登录才能获得新权限
2. **Token 刷新**：建议实现 token 刷新机制，避免频繁登录
3. **权限缓存**：前端不应缓存权限信息，应以后端校验为准
4. **日志审计**：建议记录所有权限校验失败的操作到审计日志
