## 测试流程

### 前置条件
- 后端服务运行在 `http://localhost:3000`
- 前端服务运行在 `http://localhost:5173`
- 数据库已初始化
- 至少有一个资深等级（level >= 3）的测试用户
- 至少有一个管理员（level = 4）的测试账号

### 测试步骤

#### 1. 高等级用户发布话题
```
操作：使用资深用户登录 → 点击"发布话题" → 填写表单 → 提交
预期：
- 如果内容正常：提示"话题发布成功，已通过AI审核"
- 如果内容可疑：提示"话题已提交，AI检测到可能的问题，正在等待管理员复核"
```

#### 2. AI自动审核（正常内容）
```
测试数据：
- 标题：人工智能是否会取代人类工作
- 描述：随着AI技术的发展，越来越多的工作可能被自动化替代，这是进步还是威胁？
- 分类：科技

预期：
- 话题立即通过审核（audit_status = 1）
- 在话题列表中可见
- 状态显示为"待开始"
```

#### 3. AI触发人工复核（可疑内容）
```
测试数据：
- 标题：加微信了解更多！！！
- 描述：联系方式：QQ12345，电话：1234567890

预期：
- 提示需要人工复核
- 话题不在列表中显示（audit_status = 0）
- 在复核队列中出现
```

#### 4. 管理员复核通过
```
操作：管理员登录 → 进入"内容复核管理" → 查看待复核话题 → 点击"通过"
预期：
- 话题 audit_status 更新为 1
- 话题在列表中显示
- 复核队列中移除该项
```

#### 5. 管理员复核驳回
```
操作：管理员登录 → 进入"内容复核管理" → 查看待复核话题 → 点击"驳回" → 填写原因 → 确认
预期：
- 话题 audit_status 更新为 2
- 话题不在列表中显示
- 记录用户违规
```

#### 6. 列表展示验证
```
操作：访问话题列表页面
预期：
- 只显示 audit_status = 1 的话题
- 显示分类标签（科技/社会/文化/经济）
- 显示状态标签（待开始/进行中/已结束）
- 显示发布者、参与人数、热度
- 点击卡片可跳转到话题详情
```

#### 7. 权限验证
```
测试场景：
a) 普通用户（level < 3）尝试发布话题
   预期：看不到"发布话题"按钮

b) 非管理员访问复核接口
   预期：返回 403 权限不足

c) 未登录用户访问
   预期：返回 401 未认证
```

## 关键API端点

### 话题相关
- `POST /api/debates/topics` - 创建话题（需要资深用户权限）
- `GET /api/debates/topics` - 获取话题列表（只返回 audit_status=1）
- `PUT /api/debates/topics/:topicId/audit` - 审核话题（需要管理员权限）

### 复核相关
- `GET /api/review/queue` - 获取复核队列（需要管理员权限）
- `POST /api/review/:id/approve` - 通过复核（需要管理员权限）
- `POST /api/review/:id/reject` - 驳回复核（需要管理员权限）

## 数据库验证

### 话题表（debate_topic）
```sql
-- 查看话题审核状态
SELECT id, title, audit_status, status, created_at 
FROM debate_topic 
ORDER BY created_at DESC 
LIMIT 10;

-- audit_status: 0=待审核, 1=通过, 2=拒绝
-- status: 0=待开始, 1=进行中, 2=已结束
```

### 复核队列（content_review_queue）
```sql
-- 查看待复核内容
SELECT id, content_type, status, confidence, created_at 
FROM content_review_queue 
WHERE status = 'pending' 
ORDER BY created_at DESC;
```

## 注意事项

1. **AI审核逻辑**：当前使用模拟AI，检测敏感词、特殊字符、联系方式等
2. **状态区分**：`audit_status`（审核状态）和 `status`（辩论状态）是两个独立字段
3. **权限层级**：普通用户(1) < 活跃用户(2) < 资深用户(3) < 管理员(4)
4. **列表过滤**：前端列表只显示 `audit_status = 1` 的话题

## 常见问题排查

1. **话题不显示**：检查 `audit_status` 是否为 1
2. **无法发布**：检查用户 `level` 是否 >= 3
3. **复核失败**：检查管理员权限和用户ID字段
4. **AI审核异常**：查看后端日志中的审核结果
